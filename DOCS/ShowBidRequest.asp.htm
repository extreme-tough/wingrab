Description:

Enhancements on WinGRAB application as described in field "Deliverables".

<!HR>
Platform:
Visual Basic 2008 Express
Deliverables:

I modified the XML file format of WinGRAB. Please find my current source files in the attached ZIP file. This XML file adds an “InternalLInk” for each screenshot/png file. I need to use this xml file for another application that processes the generated screenshots.

What about an external XML file settings.xml keeping the data of a configuration set (like 1000x645 screenshot), the default recording type (like window, rectangle with parameters, fullscreen), the default crop coordinates (like 0,0,1000,645) and n optional cropping sets (like Set1, 10,20,990,625 and Set2, 10,20,990,625 etc.) , the re-sizing size of the window (FormHelper.SetWindowPos(hWnd, HWND_NOTOPMOST, 0, 0, 1000, 645, 0), like 1000x645 Screenshot, 1024x768 Screenshot) and the default output folder for screenshots (value of text field folder – fullpath to folder or DESKTOP/Screenshots with DESKTOP as wildcard for UserDesktop).

I would like to add a drop-down-list „Select configuration set“ on GetDateNFolder. The first configuration set in the xml file is the default setting. After the user has selected the configuration file, all values of the selected configuration set will be used for screenshot recording.

Captured png files will be now be saved in a subfolder “png”. The xml file and the subfolder will be saved in folder with date and time like in the current release.

After stopping recording each captured png file will be cropped with default crop coordinates and copied into subfolder “runfiles”. We need a Contextmenu entry “Crop files” that crops all files in subfolder png and saves cropped file in subfolder runfiles (necessary if automatically cropping after recording has not been executed successfully).

After stopping recording a new viewer window is opened.

Window displays button bar at the top and shows the recorded png files in full window size, beginning with first png

Some buttons:

Open XML file, << (Go to first) < (Go to previous) > (Go to Next) >> (Go to Last), Delete PNG file, Switch Fullsize -> Cropped (or Cropped -> Fullsize), Crop, Edit with SnagIt

Buttons:

Open XML file: file dialog for opening the xml that has been generated by WinGRAB – displaying PNG files saved in this XML file

<< go to first png, < go to previous, > go to next, >> go to last PNG file

Delete PNG file: deletes PNG file in png and runfiles folder, deletes “InternalLink” section of this png file in XML file

Switch Fullsize -> Cropped = displays either png file from png (fullsize) or runfiles (cropped) folder

Updates current view, so user sees either fullsize or cropped window

Crop

Opens list or window with different hard coded cropping coordinates (linked to list on GetDateNFolder), user selects size, fullsize screen will be cropped again and cropped version will be saved in runfiles folder, older cropped image will be overwritten

Edit with SnagIt

Run C:\Programme\TechSmith\SnagIt 9\SnagIt32.exe "FullPathToCroppedPNGFile"

That’s it.

Please have a look at the attached source file and use this xml structure.

One more feature in WinGRAB: Could you add a Pause button while recording – click pause on contextmenu pauses recording until continueing? Probably you could set a flag and capturing is only available if this flag is not set to “Pause”.
